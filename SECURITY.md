# Security Policy

## Supported Versions

Security fixes are handled for the latest released version.

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub issues.

Use GitHub private vulnerability reporting if it is enabled for this repository. If private vulnerability reporting is not available, contact the maintainer privately through the repository owner's GitHub profile before publishing details.

Include:

- A clear description of the issue.
- Steps to reproduce or a proof of concept.
- The affected files, versions, or commit range.
- Any known impact or mitigation.

## Response Expectations

Maintainers should acknowledge valid reports within a reasonable time, coordinate fixes privately when needed, and publish details after a fix or mitigation is available.
